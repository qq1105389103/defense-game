# Unity 同内容版本规划

## 目标

Unity 版作为 HTML/Canvas 版的正式升级方向，但不替换现有 HTML 版。第一目标不是技术 Demo，而是完整复刻当前 HTML 版内容，并在性能、受击反馈、碰撞一致性和后续局域网联机上留出更好的结构。

## 内容一致性清单

- 开局流程：角色选择、难度选择、3 分钟、5 分钟、无限模式。
- 角色：原版、周贤、luo、扬、老玉。
- 初始差异：周贤攻速开局，luo 弹道开局，老玉穿透开局，扬攻击力开局，原版为标准属性。
- 基础玩法：角色只控制左右移动或触控位置，攻击自动发射。
- Boss：头部在最前，身体无限延伸，关节长度和血量有差异，断节后前方关节整体回缩并重新接上后方。
- Boss 路径：从地图上方进入，横向推进并同时缓慢下移，最终压迫到玩家区域。
- 关节：普通、脆弱、装甲、高血量、奖励关节等类型保持现有规则。
- 奖励：面前两个奖励箱、奖励关节掉落二选一、救急奖励、普通奖励、稀有奖励。
- 稀有奖励：影分身、穿透、弹道、暴击等现有规则继续保留，不设置爽感上限的奖励仍保留。
- 胜负：限时模式撑到时间胜利，无限模式持续计分，Boss 压到玩家区域失败。
- 数值：以 HTML 版 0.0.3 平衡参数作为 Unity 第一版默认参数。

## Unity 架构建议

- 使用 Unity 2D 项目，目标平台优先 PC + Android 浏览/安装包测试。
- `GameConfig` 使用 ScriptableObject 保存角色、奖励、Boss、难度、模式和 DPS 平衡参数。
- `GameController` 管理开局选择、计时、胜负、模式状态。
- `PlayerController` 管理输入、角色属性、自动开火和奖励应用。
- `BossController` 管理路径推进、无限补尾、断节回缩、触底判定。
- `BossSegment` 表示一个可受击关节，保存血量、长度、类型、奖励标记和碰撞体。
- `BossPlateView` 只负责关节片视觉拼接，不参与血量逻辑。
- `RewardSystem` 管理面前奖励箱、奖励关节掉落、救急奖励和二选一领取。
- `ProjectileSystem` 使用对象池管理子弹，真实伤害和视觉子弹分离。
- `FloatingTextPool`、`HitEffectPool`、`ExplosionPool` 使用对象池，避免中后期 GC 卡顿。

## 视觉与碰撞

- Boss 头和关节片沿用现有 `resource` 素材，导入 Unity 后切成 Sprite Atlas。
- 每个 Boss 关节使用一组同类关节片平移排列，转弯时不让单片独立旋转翻转。
- 关节碰撞使用 BoxCollider2D 或 CapsuleCollider2D，长度必须和视觉显示一致。
- 子弹判定优先命中视觉对应的当前关节，避免打到接触位却扣前一节血的问题。
- 受击反馈使用 SpriteRenderer 颜色、材质参数或预制受击材质实现深紫短闪，不遮挡血量。
- 暴击只改变伤害字体颜色、粗细、缩放曲线，不显示“暴击”文字。

## 性能目标

- 大量子弹、影分身、飘字和 Boss 关节中后期仍保持稳定帧率。
- 所有高频对象使用对象池，不在战斗帧频繁 Instantiate/Destroy。
- Boss 关节位置使用路径采样缓存，只有 Boss 推进、断节、补尾时更新必要段。
- UI 血量文字尽量使用 TextMeshPro 对象池，避免每帧创建文本。
- 移动端优先控制 overdraw、透明粒子数量和 SpriteRenderer 数量。

## 联机方向

- 第一阶段先完成单机同内容版本。
- 第二阶段做局域网多人合作，最多 4 人。
- 联机仍采用服务端权威：Boss、伤害、奖励、胜负由权威端决定。
- 客户端只发送输入、角色选择和准备状态。
- 子弹、飘字、受击、爆炸可以本地表现，但命中结果以权威端同步为准。
- Boss 血量和移动速度按人数系数调整，初始使用 HTML 版人数平衡作为参考。

## 阶段验收

1. 单机可玩：角色、Boss、奖励、胜负闭环完整。
2. 内容对齐：逐项核对 HTML 版玩法清单，不缺功能。
3. 数值对齐：普通难度 3 分钟节奏接近 HTML 0.0.3。
4. 性能验证：中后期大量关节、弹道、分身、飘字不卡顿。
5. 手机验证：触控移动顺手，核心 UI 不遮挡战斗区域。
6. 联机验证：2 到 4 人局域网同屏合作，Boss 血量、奖励、胜负一致。

## 当前限制

当前机器没有检测到可调用的 Unity Editor，因此本次只落地 Unity 同内容迁移规格。安装 Unity 后，再新建独立工程并按本文档实现，不覆盖 HTML 版。
