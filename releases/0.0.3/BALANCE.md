# 数值平衡记录

## 2026-06-14 稀有强奖励降频

- 问题：
  - 影分身和穿透太强，如果稀有出现频率和奖励关节密度都偏高，会很容易滚成无难度局。
  - 需要保留稀有带来的爽感，但降低连续获得强稀有的概率。
- 改动：

```js
rewardChance = 0.16
奖励关节固定间隔：index % 11 -> index % 13
frontRareChance = 0.035
jointRareChanceStart = 0.16
jointRareChanceMin = 0.07
```

- 稀有池改为加权抽取：

```js
影分身 rareWeight = 0.25
穿透 rareWeight = 0.45
暴击 rareWeight = 1.60
```

- 设计意图：
  - 奖励关节略少，减少奖励链滚雪球。
  - 稀有整体少一点，且影分身、穿透比暴击更少见。
  - 影分身/穿透仍然无限叠，只是更难连续拿到。

## 2026-06-13 稀有递减与后期性能

- 问题：
  - 影分身等稀有奖励需要无限叠，不能设置硬上限。
  - 奖励关节第一个稀有概率应该高一些，拿得越多概率越低，但不能低到完全没有。
  - 后期影分身、攻速、弹道都很高时，真实生成大量子弹、飘字和粒子会导致卡顿。
- 改动：

```js
jointRareChanceStart = 0.22
jointRareChanceDecay = 0.06
jointRareChanceMin = 0.10
frontRareChance = 0.055
maxVisualShooters = 5
```

- 稀有概率规则：
  - 奖励关节稀有概率从 `22%` 开始。
  - 每拿到 1 个稀有，概率降低 `6%`。
  - 最低不低于 `10%`。
  - 正面箱子按 `5.5%` 明确从稀有池抽取，不再被全池稀释。
- 性能优化：
  - 影分身实际无限叠。
  - 屏幕上最多显示 `5` 组发射器。
  - 超出的影分身折算为子弹伤害倍率，不损失实际输出。
  - 飘字和粒子增加数量上限，避免高攻速后特效堆积卡顿。
- 设计意图：
  - 稀有越叠越爽，但不靠无限创建子弹对象实现。
  - 早期更容易感受到稀有惊喜，后期仍保留最低概率。

## 2026-06-13 移动压迫替代血墙

- 目标：
  - BOSS 持续压迫应主要来自移动速度，而不是关节全都打不动。
  - 关节要保持较高击破率，玩家不断看到击破、断节、奖励的反馈。
  - 稀有奖励略微提高概率，避免整局完全没有。
- 改动：

```js
easy.speed = 1.10
normal.speed = 1.18
hard.speed = 1.50
speedBase = 34
speedRamp = 0.50
```

- 血量基准同步降低：

```js
demandRatio = 0.44
timeHpRamp = 0.34
minHp = 12000
weakMinHp = 6200
rewardMinHp = 8200
```

- 关节目标击破时间下调：

```js
weakKillTime = [3.4, 5.2]
crackKillTime = [1.8, 3.4]
normalKillTime = [4.8, 7.6]
toughKillTime = [10, 17]
wallKillTime = [28, 54]
rewardKillTime = [2.8, 6.8]
armoredKillTime = [9, 15]
```

- 稀有概率略提：

```js
frontRareChance = 0.055
jointRareChance = 0.20
```

- 设计意图：
  - 大多数普通关节应该能被持续击破。
  - Boss 仍会因为速度更快而逼近底线，形成持续压力。
  - 稀有奖励更容易在一局内出现，但仍不会进入 Boss 血量参考 DPS。

## 2026-06-13 稀有不参与 Boss 参考 DPS

- 问题：
  - 稀有奖励概率偏高，导致稀有不够“惊喜”。
  - 虽然预期 DPS 模型已经排除了稀有奖励，但 Boss 定标里的真实 DPS 参考仍可能吃到分身、穿透、暴击等稀有属性，造成拿稀有一段时间后 Boss 血量暴涨。
- 改动：

```js
frontRareChance = 0.03
jointRareChance = 0.14
```

- 新增稀有来源计数：

```js
rareClones
rarePierce
rareCrit
```

- Boss 参考 DPS 改为 `bossReferenceDps()`：
  - 计算 Boss 血量时会扣除稀有奖励提供的分身、穿透、暴击。
  - 实际战斗 DPS 仍然完整吃稀有奖励。
  - 角色自带属性不算稀有，仍参与参考 DPS。
- 设计意图：
  - 稀有奖励只降低难度、制造爽感，不反向推高 Boss 血量。
  - 稀有概率降低，让拿到时更像事件和惊喜。

## 2026-06-13 普通奖励作为通关基线

- 问题：
  - 普通难度不应该变成“没有稀有就打不死”。
  - 稀有奖励应该是大幅降低难度、制造爽感的额外惊喜，而不是通关硬门槛。
- 改动：

```js

```

- BOSS 血量预期模型调整：
  - `simulatedFullRewardDps()` 只使用非稀有奖励计算通关基线。
  - 奖励关节在模型里也只按普通高档奖励计算。
  - 稀有奖励不再进入 Boss 血量成长预期。
- 稀有奖励取消上限：

```js

maxPierce = Infinity
maxCrit = Infinity
```

- 普通奖励略增强：


- 关节击杀时间继续下调：

```js

```

- 设计意图：
  - 普通奖励路线应该能支撑普通难度通关。
  - 稀有奖励越多越爽，能够大幅降低难度。
  - 运气特别好时允许玩家打出明显碾压感，不再被 Boss 血量定标吃掉。


- 稀有奖励增强：


- 设计意图：
  - 普通难度允许好运玩家明显滚雪球。
  - 困难难度仍保留高压和运气需求。
  - 非影分身稀有奖励也要有足够强度，避免“拿了一堆稀有还是没用”的挫败感。

## 2026-06-13 后期低血量入口

- 问题：
  - 中后期如果连续刷高血量关节，玩家一旦打不掉，就同时失去奖励关节、正面箱子和击退空间。
  - 这种失败不够有选择，容易变成纯堵死。
- 改动：

```js
timeHpRamp = 0.62
crackKillTime = [2.2, 4.2]
crackChanceStart = 55
crackChanceMax = 0.16
weakMinHp = 9500
rewardMinHp = 12000
```

- 新增“裂缝关节”：
  - 55 秒后开始小概率出现。
  - 概率随时间慢慢升高，最高约 `16%`。
  - 不会替代固定装甲位。
  - 击杀目标时间很短，作为后期火力入口和喘息点。
  - 颜色偏绿色，方便玩家发现。
- 设计意图：
  - 中后期仍然要难，但不能每一节都是硬墙。
  - 玩家被压迫时，偶尔看到低血量段可以选择转火突破。
  - DPS 需求成长稍微放缓，让好运和正确选择有更长的发挥窗口。

## 2026-06-13 幸运爆发保留爽感

- 问题：
  - 运气好拿到影分身、弹道、穿透等强奖励时，玩家应该明显爽一段时间。
  - 之前 Boss 血量虽然不是完全按当前 DPS 追，但仍会较快吃掉高 DPS 带来的优势。
- 改动：

```js
demandRatio = 0.70
expectedRewardQuality = 0.66
actualDpsWeightLow = 0.32
actualDpsWeightHigh = 0.10
actualDpsClamp = [0.45, 1.22]
rewardJointModelInterval = 32
```

- 新公式：
  - 玩家 DPS 低于预期时，Boss 定标会参考 `32%` 真实 DPS，避免发育差时完全被模型压死。
  - 玩家 DPS 高于预期时，Boss 定标只参考 `10%` 真实 DPS，保留幸运奖励带来的碾压窗口。
  - 真实 DPS 对 Boss 血量的最高影响限制在预期的 `1.22` 倍以内。
- 设计意图：
  - 欧的时候要爽，不能刚变强就被 Boss 血量马上追上。
  - 但长期不拿奖励仍然会被时间成长、难度倍率和 Boss 推进速度压住。

## 2026-06-13 后期奖励节奏与箱子可达性

- 问题：
  - 之前 Boss 血量预期按固定奖励节奏估算，默认大约每 `8` 秒获得一次奖励。
  - 真实游戏里，后期正面奖励箱需求成长后，击破时间会变长，DPS 成长跟不上模型预期。
  - Boss 压迫到玩家前方时，子弹会先打到 Boss，导致画面上有箱子但玩家很难打到箱子。
- 改动：
  - `simulatedFullRewardDps()` 改为模拟真实箱子节奏：
    - 两个正面箱子分别计算冷却。
    - 箱子需求随 `scaleGrowth` 增长。
    - 攻速开箱收益按 `speedProgressDecay` 递减。
    - 弹道和影分身会提高开箱命中数。
    - 奖励关节按期望间隔额外提供一次高质量成长。
  - 新增估算参数：

```js
modelAttention = 0.72
rewardJointModelInterval = 22
```

  - `modelAttention` 表示玩家不可能 100% 时间都在完美打箱子，需要在 Boss 和箱子之间分配火力。
  - `rewardJointModelInterval` 表示奖励关节也会参与后期成长预期，避免只靠正面箱子计算。
  - 增加“箱子优先通道”：当子弹正朝奖励箱所在竖向路径飞行时，不会先被 Boss 身体截掉。
- 设计意图：
  - 后期 Boss 血量要求更贴近真实奖励获取速度。
  - 玩家在 Boss 压上来时仍能选择冒险打箱子，而不是被迫完全失去成长入口。
  - 奖励箱仍然需要玩家主动站位和分配火力，不是免费获得。

## 0.0.3 数值平衡方案

### 总目标

- 难度：偏高。不持续拿奖励、不做相对正确的奖励选择，很难通关。
- 爽度：奖励获取不能太难，玩家应该频繁感到变强。
- 核心压力：Boss 血量和移动速度持续制造压迫，奖励负责提供成长曲线。
- 项目结构：先保持单文件原型，但所有核心数值集中到 `BALANCE` 配置对象，便于反复测试修改。

### 项目结构优化

- 新增 `BALANCE` 配置区，集中管理：
  - `player`：初始攻击、基础攻速、一命制、最大弹道。
  - `boss`：血量、长度、宽度、移动速度。
  - `rewardBox`：奖励箱基础需求、成长、攻速递减。
  - `upgrades`：奖励类型、数值、成本、评分。
- 暂不拆分 JS 文件，避免当前快速试玩阶段增加构建或引用复杂度。
- 后续规则稳定后再拆为：

```text
index.html
src/game.js
src/balance.js
BALANCE.md
```

### 难度与模式

- 开局可选难度：

```js
简单：demand 0.7，speed 1.05，ramp 1.00，救急线 H - 420
普通：demand 0.95，speed 1.22，ramp 1.18，救急线 H - 340
困难：demand 1.25，speed 1.42，ramp 1.35，救急线 H - 260
```

- 开局可选模式：

```js
3分钟：duration 180，hp 1.00，speed 1.00
5分钟：duration 300，hp 1.35，speed 1.08
无限：duration Infinity，hp 1.55，speed 1.16
```

- 设计意图：
  - 简单以当前测试手感为基准，但 Boss 压迫速度略提高。
  - 普通要求稳定拿奖励并做较好选择。
  - 困难需要高质量奖励路线和一定随机运气。
  - 5分钟和无限模式会额外提高血量与速度，避免时间越长越无聊。

### Boss 关节血量

- 玩家生命固定为 `1`，不再通过奖励增加生命。

- 当前参数：

```js
weakChance = 0.28
toughChance = 0.18
wallChance = 0.07
rewardChance = 0.18

activateDistance = -260
demandRatio = 0.7
fullRewardInterval = 8
expectedRewardQuality = 0.72
actualDpsWeight = 0.4
actualDpsClamp = [0.45, 1.8]
timeHpRamp = 0.85
minHp = 18000

weakKillTime = rand(4.2, 6.2)
normalKillTime = rand(7, 12)
toughKillTime = rand(16, 28)
wallKillTime = rand(42, 85)
rewardKillTime = rand(4, 14)
armoredKillTime = rand(18, 26)
headKillTime = rand(90, 140)
```

- 设计意图：
  - 不再使用固定生成血量。固定血量会在玩家成长后变成“砍瓜切菜”。
  - Boss 血量不再完全按当前 DPS 定标，避免玩家觉得“成长没意义”。
  - Boss 使用“满奖励期望 DPS × 难度比例”为主，真实 DPS 只占 `40%`：

```js
fullRewardDps = 模拟每 8 秒拿一次奖励，并按 72% 高质量选择计算
expectedDps = max(startDps * 7, fullRewardDps * 0.7)
actualDps = clamp(playerDps, expectedDps * 0.45, expectedDps * 1.8)
targetDps = expectedDps * 0.6 + actualDps * 0.4
hp = targetDps * (1 + elapsedRatio * 0.85) * targetKillTime
```

  - 关节初始生成时会有一个目标击杀时间，等它接近屏幕时按当下 DPS 重算一次血量。
  - `activateDistance = -260` 表示关节还在屏幕外上方时完成定标，避免玩家看到血量突然变化。
  - 开局保底按约 `7` 倍初始 DPS 预期计算，避免初始血量太低。
  - 即使玩家摆烂不成长，预期 DPS 和 `timeHpRamp` 也会让 Boss 随时间变硬。
  - 玩家真实 DPS 有影响，但只有 `40%` 权重并且有上下限，避免成长被完全抵消。
  - 正常进度下，脆弱关节目标抗约 `5` 秒，装甲关节目标抗约 `20` 秒。
  - 脆弱/普通关节能被持续击破，提供反馈。
  - 少量墙级关节目标击杀时间很长，负责持续推进压迫。
  - 奖励关节概率提高，且不一定很硬，用作“突然救急”的爽点。
  - 装甲关节是高压段，需要奖励成长后才能稳定处理。
  - 头部血量最高，击破头部属于强反馈事件。
- 后续调参参考：
  - 如果太简单：提高 `demandRatio`、`timeHpRamp`、`wallChance` 或对应 `KillTime`。
  - 如果太堵：降低 `wallChance` 或 `wallKillTime`，不要先削弱普通/脆弱关节。
  - 如果爽感不够：提高 `weakChance` 或降低 `weakKillTime`。
  - 如果奖励关节太难拿：降低 `rewardKillTime`。

### 奖励数值

- 当前奖励表：

```js
攻击 +120：cost 20，score 2
攻击 +260：cost 30，score 5
攻击 +620：cost 48，score 8
攻速 x2：cost 36，score 6，攻速倍率 +2
攻速 x3：cost 52，score 8，攻速倍率 +3
多一弹道：cost 68，score 10，弹道 +1，最大 5
击退 +180：cost 24，score 3
```

- 设计意图：
  - 弹道最强，因为对 Boss 和奖励箱都没有递减，所以成本和评分最高。
  - 攻速是加算倍率，不做乘法爆炸。
  - 攻击奖励分为小/中/大三档，保证选择有差异。
  - 击退奖励是一命制下的保命奖励，能救急但不直接提高生命。

### 奖励箱难度

- 当前参数：

```js
startCosts = [20, 24]
startScale = 0.72
scaleGrowth = 0.12
maxScale = 2.8
speedDemandScale = 0.08
speedProgressDecay = 0.38
respawnDelay = 7
emergencyPressureY = 按难度配置
emergencyCooldown = 18
```

- 设计意图：
  - 奖励箱不难拿，保证爽感。
  - 领取奖励后常驻箱子消失 `7` 秒，期间不显示、不碰撞、不挡弹道。
  - 消失窗口让玩家有时间把火力打到 Boss 上，避免一直站在箱子前无脑刷奖励。
  - Boss 压迫线过近时，前方随机位置会刷一个救急奖励方块。
  - 简单更早刷救急，困难更晚刷救急。
  - 救急奖励方块倾向击退或高价值输出，打掉后直接消失，不原地冷却。
  - 开箱需求会逐步成长，避免后期无脑刷。
  - 攻速对开箱有效，但有递减。
  - 多重弹道不递减，保持“弹道奖励很爽”的特点。
- 后续调参参考：
  - 如果奖励节奏太慢，降低 `respawnDelay` 到 `5-6` 秒。
  - 如果仍然无脑刷奖励，提高 `respawnDelay` 到 `9-10` 秒或提高 `scaleGrowth`。

### Boss 关节长度

- 当前参数：

```js
headLen = 330
normalLen = rand(170, 330) + min(targetKillTime, 8) * 72
rewardLen = rand(300, 500)
armoredLen = rand(420, 640)
```

- 设计意图：
  - 最长装甲关节可达到半个游戏界面高度左右，制造巨大身体压迫感。
  - 目标击杀时间更长的普通关节会更长，但仍保留随机差异。
  - 最短普通关节接近之前手感，不让整条 Boss 都过长。

### Boss 移动速度

- 当前参数：

```js
speedBase = 28
speedRamp = 0.42
speedRampTime = 120
```

- 实际速度公式：

```js
speed = speedBase + Math.min(speedRampTime, elapsedTime) * speedRamp
```

- 设计意图：
  - 前期给玩家拿第一轮奖励的时间。
  - 中后期 Boss 明显加压，不持续拿奖励会被推到底线。
- 最大速度约 `28 + 120 * 0.42 = 78.4` 路径单位/秒。

### 地狱难度测试估算

- 估算路线：
  - 满奖励期望路线：每 `8` 秒拿一次奖励，约 `72%` 选到高质量输出奖励。
- 满奖励期望路线下，不同模式和难度的后期装甲等效击杀时间估算：

```text
3分钟 简单：约 26 秒
3分钟 普通：约 35 秒
3分钟 困难：约 44 秒

5分钟 简单：约 36 秒
5分钟 普通：约 48 秒
5分钟 困难：约 59 秒

无限 简单：约 41 秒
无限 普通：约 55 秒
无限 困难：约 68 秒
```

- 设计判断：
  - 只有持续拿奖励且选择偏正确，才可能处理后期装甲/墙级关节。
  - 满奖励路线仍然能明显变强，但 Boss 会按 `70%` 满奖励期望 DPS 定标，并随时间大幅加压。
  - 困难模式通关需要奖励路线、Boss 关节随机分布和救急时机都比较好。

### 后续观察点

- 如果玩家不拿奖励也能撑很久：提高 `speedBase`、`speedRamp` 或 Boss 血量。
- 如果奖励拿太多仍然打不动：降低 `rewardKillTime`、`armoredKillTime` 或 `wallKillTime`。
- 如果奖励太容易滚雪球：提高 `scaleGrowth` 或降低高档攻击奖励。
- 如果弹道过强：降低 `maxShots`，或把弹道纳入奖励箱需求缩放。

## 2026-06-13

### Boss 关节长度差异

- 问题：Boss 关节长度过于接近，并且整体偏短，看起来像短珠子串，不像无穷长的大型 Boss。
- 改动：按关节类型设置不同长度范围。
  - 头部：固定长关节。
  - 装甲关节：最长范围。
  - 奖励关节：中长范围。
  - 普通关节：比之前更大的随机范围。
- 后续调参参考：
  - 如果 Boss 仍然像珠子串，继续提高 `len` 范围。
  - 如果身体过稀、路径难读，降低 `len` 范围或增加绘制连接厚度。

### 奖励方块进度

- 问题：固定受击冷却让攻速提升对奖励箱的收益太生硬。
- 改动：取消固定冷却。每颗命中奖励箱的子弹都会推进奖励进度。
- 攻速对奖励箱有收益递减：

```js
progressPerBullet = 1 / Math.pow(totalAttackSpeedMultiplier, 0.45)
```

- 高攻速仍然能更快开箱，但不会线性爆炸。
- 多重弹道不递减。每条弹道都完整结算一次 `progressPerBullet`。
- 奖励箱需求会随攻速略微提高：

```js
demandScale = 1 + (totalAttackSpeedMultiplier - 1) * 0.1
```

- 后续调参参考：
  - 高攻速开箱太快：提高 `0.45` 或 `0.1`。
  - 攻速对开箱太弱：降低 `0.45` 或 `0.1`。
  - 除非多重弹道过强，否则不要把 `shots` 计入需求缩放。

### Boss 受击面积与 8 弹道覆盖

- 问题：关节和头部面积偏小，多弹道时不能稳定命中同一段身体，弹道提升的反馈不够明显。
- 目标：Boss 身体段的受击面积至少能容纳约 8 个弹道的横向攻击覆盖。
- 当前弹道间距：

```js
spread = 22
```

- 8 弹道横向中心跨度约为：

```text
(8 - 1) * 22 = 154
```

- 上一版参数，已被后续“Boss 长度与受击判定拆分”覆盖：

```js
head len = 220
armored len = rand(260, 380)
reward len = rand(230, 340)
normal len = rand(180, 300)

head width = 190
armored width = rand(164, 210)
reward width = rand(152, 196)
normal width = rand(136, 176)
```

- 说明：普通关节宽度最低 136，配合子弹半径和身体胶囊判定，已经接近 8 弹道覆盖；奖励/装甲/头部能更稳定承接多弹道。
- 后续调参参考：
  - 如果 8 弹道仍有明显漏弹，提高 `normal width` 下限到 `150`。
  - 如果 Boss 过胖导致躲避和目标选择消失，降低 `normal width` 上限或只提高头/特殊关节宽度。

### Boss 长度与受击判定拆分

- 问题 1：关节长度虽然变长，但仍然偏均匀，缺少“血多的关节更厚重、更长”的直觉。
- 问题 2：受击判定使用两个关节中心之间的完整连接线，导致上一节的受击区域覆盖到下一节关节。
- 改动：
  - 普通关节先随机血量权重 `normalScale = rand(0.75, 1.85)`。
  - 普通关节长度增加一部分血量权重影响：

```js
normal len = rand(160, 250) + normalScale * 42
```

  - 装甲/奖励/头部关节继续保持更长的长度范围：

```js
head len = 250
armored len = rand(290, 430)
reward len = rand(250, 380)
```

  - 历史方案：受击判定不再吃完整中心连接线，而是只吃当前关节到下一关节方向的前 `62%`。这个方案仍会在接触位产生视觉归属歧义，已被下面“视觉一致命中归属”替换。

```js
hit = pointToLimitedSegmentDistance(bullet, currentPoint, nextPoint, 0.62)
```

- 目的：
  - 血量高的普通关节更容易表现为长关节。
  - 子弹打到下一节附近时，不会被上一节提前吃掉。
- 后续调参参考：
  - 如果仍然覆盖到下一节，把 `0.62` 降到 `0.55`。
  - 如果关节之间出现明显判定空白，把 `0.62` 提到 `0.68` 或略增宽度。
  - 如果长度差异还不明显，提高 `normalScale * 42` 里的 `42`。

### 视觉一致命中归属

- 问题：连接带看起来是某一节的颜色，但命中时可能扣前一节血量。典型情况是子弹打在黄色连接带接触位，却扣了前面红色关节血。
- 改动：
  - 命中判定先检测每个关节本体椭圆区域。
  - 再检测两个关节中心之间的连接带。
  - 连接带按中点分成两半：
    - 前半段归前一节。
    - 后半段归后一节。
  - 绘制也改成两半纯色，不再使用渐变连接带。
- 目的：玩家看到黄色就扣黄色，看到红色就扣红色，命中反馈和视觉一致。
- 后续调参参考：
  - 如果接缝处仍然难判断，可以把中点分割改成按宽度或血量偏移。
  - 如果关节本体太容易抢判定，可以降低椭圆判定的 `1.15` 容差。

## 2026-06-13 地图、奖励来源与稀有奖励调整

### 地图扩大

- 画布逻辑尺寸从 `720 x 1280` 调整为 `1440 x 1280`，战场面积约为原来的 `2` 倍。
- 目的：
  - Boss 横向爬行时，同屏能容纳更多关节。
  - 玩家可以选择攻击不同身体段，而不是只盯着正前方。
  - 穿透、弹道、影分身这类奖励在宽图中有更明确的价值。
- Boss 初始生成关节数从 `72` 提高到 `104`，最低维持数从 `78` 提高到 `112`，继续保持“身体无穷无尽”的压迫感。

### 正面奖励箱削弱为稳定成长来源

- 正面两个奖励箱的需求上调：

```js
startCosts = [34, 40]
startScale = 0.9
scaleGrowth = 0.18
maxScale = 3.8
speedDemandScale = 0.13
speedProgressDecay = 0.46
```

- 设计意图：
  - 正面箱子仍然值得打，但不能成为唯一正确策略。
  - 攻速提高后开箱仍会变快，但衰减更明显。
  - 每次获得正面奖励后，后续需求增长更快，逼迫玩家把火力转向 Boss 和奖励关节。
  - 正面箱子只有约 `12%` 概率刷到稀有奖励，避免玩家一直蹲箱子轻松滚雪球。

### 奖励关节增强为主要成长入口

- 奖励关节概率提高：

```js
rewardChance = 0.28
```

- 奖励关节击破时间降低：

```js
rewardKillTime = rand(3.5, 9)
```

- 设计意图：
  - 奖励关节更常见，而且不一定很硬。
  - 玩家即使 DPS 没有明显提升，也有机会击破奖励关节获得救急或翻盘奖励。
  - 困难模式下，通关更依赖奖励关节的选择和运气，而不是无脑刷正面箱子。
  - 奖励关节掉落的二选一奖励有约 `36%` 倾向出现稀有奖励。

### 新增稀有奖励

```js
影分身：复制一个同攻击力、攻速、弹道数的攻击个体，最多 2 个
穿透 +1：子弹可额外命中不同 Boss 关节，最多 2 层
暴击 +15%：子弹有概率造成 2 倍伤害，最多 75%
```

- 设计意图：
  - `影分身` 是最高价值翻盘奖励，直接复制当前输出体系，越后期越强。
  - `穿透` 适合宽地图和多关节场景，能把火力从单点扩展到身体链条。
  - `暴击` 是稳定放大项，不像影分身那么爆炸，但能持续提高击破概率。
  - 困难模式需要这些高波动奖励参与，形成“又爽又难”的通关曲线。

### Boss 血量需求同步微调

```js
demandRatio = 0.76
expectedRewardQuality = 0.78
```

- Boss 血量现在会把稀有奖励纳入满奖励期望 DPS 估算。
